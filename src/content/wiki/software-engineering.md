Software engineering is not a single skill but a cluster of concerns: how code is structured, how it is verified, how it is maintained, and how the humans writing it learn and communicate. The sources here cut across all of those layers, and the most persistent tension running through them is whether AI-assisted workflows strengthen or erode the underlying craft.

On design and structure, several pieces argue for restraint. [AI Likes Deep Modules](/reading/2026-05/2026-05-04t231343-ai-likes-deep-modules) makes the case that small interfaces hiding large implementations reduce complexity for humans and LLMs alike, and that shallow module proliferation is the more common failure mode. [Single Responsibility, the Distorted Principle](/reading/2026-06/2026-06-04t073318-single-responsibility-the-distorted-principle) pushes back on the popular interpretation of SRP as "do only one thing," arguing that over-granularizing classes violates the cognitive simplicity the principle was meant to protect. [A Better Way to Build Angular Components](/reading/2026-04/2026-04-30t232001-a-better-way-to-build-angular-components-from-inputs-to) extends this to component design: inputs-bloated components should be decomposed via the Composite Components pattern so each concern stays encapsulated.

On testing, the picture is complicated by AI's growing role in generating tests that look correct but are not. [Code Smells when you get AI to write your Frontend Tests](/reading/2026-06/2026-06-22t185420-code-smells-when-you-get-ai-to-write-your-frontend-tests) catalogs over twenty recurring problems — over-mocking, happy-path bias, tests written to match a buggy implementation rather than intended behavior. [How AI Code Review Can Make Correct Code Worse](/reading/2026-06/2026-06-23t212958-how-ai-code-review-can-make-correct-code-worse) finds that agent-driven implementer-reviewer-fixer pipelines cause weaker fixer models to overreach beyond review scope and break correct code, with softer instructions as the only discovered mitigation. [Designing Playwright Tests That Survive UI Refactors](/reading/2026-05/2026-05-05t135218-designing-playwright-tests-that-survive-ui-refactors) argues the root cause of brittle test suites is coupling to implementation details rather than semantic roles and accessible names. [What CI Actually Looks Like at a 100-Person Team](/reading/2026-04/2026-04-30t195531-what-ci-actually-looks-like-at-a-100-person-team) shows the other side: at scale, an AI triage agent processing 575K weekly jobs can trace flaky tests to root causes and open fix PRs automatically.

On the question of whether AI is net-positive for engineering quality, the sources split sharply. [Agentic Coding is a Trap](/reading/2026-04/2026-04-27t145041-agentic-coding-is-a-trap) argues that full agentic workflows accelerate skill atrophy and invert priorities toward speed over understanding, advocating instead for LLMs as secondary delegation tools. [The Perils of AI to the Software Engineering Profession](/reading/2026-05/2026-05-14t223612-the-perils-of-ai-to-the-software-engineering-profession) makes the safety-critical case: shipping AI-generated code without review is categorically incompatible with nuclear or flight-control systems. [The Coming Loop](/reading/2026-06/2026-06-23t161552-the-coming-loop) warns that harness loops orchestrating agents amplify LLMs' worst tendencies and risk producing codebases that require machine participation to maintain. [When Code Is Cheap, Does Quality Still Matter?](/reading/2026-05/2026-05-22t091746-when-code-is-cheap-does-quality-still-matter) answers yes: LLMs generate polished technical debt faster than any individual engineer, so taste and judgment remain the binding constraint. Countering the pessimism somewhat, [Formal Methods and the Future of Programming](/reading/2026-06/2026-06-15t021106-formal-methods-and-the-future-of-programming) argues that agentic coding has made formal verification newly cost-effective by lowering the cost of writing proofs.

On tacit knowledge and expertise, [The Tacit Dimension](/reading/2026-05/2026-05-19t110710-the-tacit-dimension-why-your-best-engineers-cant-tell-you) argues, drawing on Polanyi, that the most valuable engineering expertise — pattern recognition, design intuition, unwritten conventions — is structurally inaccessible to AI and can only be transmitted through apprenticeship. [Why Senior Developers Fail to Communicate Their Expertise](/reading/2026-05/2026-05-13t060018-why-senior-developers-fail-to-communicate-their-expertise) adds a communication layer: senior engineers speak in complexity-management terms while the rest of the organization thinks in uncertainty-reduction terms, and bridging that gap is the real challenge of expertise.

On hiring and interviews, [Learn Algorithms for Interviews, Forget Them for Work](/reading/2026-04/2026-04-30t155134-learn-algorithms-for-interviews-forget-them-for-work) argues that algorithm tests weakly correlate with production performance, where real engineering means reading tradeoffs and shipping incrementally against messy, unbounded inputs. [The Systemic Decay of Tech Hiring](/reading/2026-06/2026-06-22t182141-the-systemic-decay-of-tech-hiring) traces how error asymmetry drives ever-harder tests and candidates overfit via Goodhart's Law. [Your Onboarding Is a Hazing Ritual](/reading/2026-05/2026-05-08t112608-your-onboarding-is-a-hazing-ritual-and-you-call-it-agile) extends the critique to onboarding: agile labels applied to packed meeting calendars and same-sprint workloads systematically set new hires up to fail.

On supply-chain security, [Supply-chain attack using invisible Unicode code](/reading/2026-04/2026-04-30t231634-supply-chain-attack-using-invisible-code-hits-github-and) describes 151 malicious npm packages encoding payloads in invisible Unicode variation-selector characters, undetectable by code reviewers and static analysis. [Approaching Zero Bugs](/reading/2026-05/2026-05-02t094735-approaching-zero-bugs) uses curl's vulnerability data to argue there is no measurable sign yet that AI-assisted static analysis is reducing latent bugs in open-source projects.

Practical craft articles fill out the picture: [The Unwritten Laws of Software Engineering](/reading/2026-06/2026-06-10t073045-the-unwritten-laws-of-software-engineering) distills production-incident lessons including rolling back before debugging and treating every external dependency as a future outage. [The Git Commands I Run Before Reading Any Code](/reading/2026-06/2026-06-18t024208-the-git-commands-i-run-before-reading-any-code) and [How I Audit a Legacy Rails Codebase](/reading/2026-06/2026-06-18t090801-how-i-audit-a-legacy-rails-codebase-in-the-first-week) both argue that understanding a codebase starts with its social and historical record, not its files. [When Impressive Performance Gains Do Not Matter](/reading/2026-06/2026-06-30t185207-when-impressive-performance-gains-do-not-matter) cautions that order-of-magnitude optimizations often fail to change outcomes due to attention thresholds and pipeline backpressure.
