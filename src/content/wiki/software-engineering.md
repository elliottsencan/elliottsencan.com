Software engineering is not reducible to writing code. The sources here collectively argue that the field involves at least three overlapping concerns: technical craft (how code is structured and executed), process and tooling (how work is organized and reviewed), and professional judgment (what gets built, why, and by whom). Each layer has been destabilized in distinct ways by the current wave of AI coding tools, but the underlying tensions predate that shift.

On technical craft, the recurring theme is that clarity costs less than it appears. [AI Likes Deep Modules](/reading/2026-05/2026-05-04t231343-ai-likes-deep-modules) argues that small interfaces hiding large implementations reduce complexity for both humans and LLMs. [Single Responsibility, the Distorted Principle](/reading/2026-06/2026-06-04t073318-single-responsibility-the-distorted-principle) makes the complementary point that SRP is about cohesion under a single accountability, not decomposition for its own sake. Both push back against the instinct to make code smaller at the cost of making it harder to reason about. The performance side of craft is equally unforgiving: [5x faster fast_blur in image-rs](/reading/2026-05/2026-05-14t151252-5-faster-fastblur-in-image-rs) shows how profiling and arithmetic substitution can yield a 5.9x speedup, while [When Impressive Performance Gains Do Not Matter](/reading/2026-06/2026-06-30t185207-when-impressive-performance-gains-do-not-matter) reminds that optimization is irrelevant if the bottleneck is elsewhere in a pipeline.

Process and tooling questions run throughout. [What CI Actually Looks Like at a 100-Person Team](/reading/2026-04/2026-04-30t195531-what-ci-actually-looks-like-at-a-100-person-team) describes a real-world AI triage agent handling 575K weekly CI jobs, opening fix PRs for flaky tests automatically. [Designing Playwright Tests That Survive UI Refactors](/reading/2026-05/2026-05-05t135218-designing-playwright-tests-that-survive-ui-refactors) and [Playwright Testing in Staging vs Production](/reading/2026-05/2026-05-15t120337-playwright-testing-in-staging-vs-production) address test suite design at the level of what tests should couple to: semantic roles and accessible names rather than DOM structure. [The Unwritten Laws of Software Engineering](/reading/2026-06/2026-06-10t073045-the-unwritten-laws-of-software-engineering) distills operational wisdom from production incidents, including the standing instruction to roll back before debugging. [How to Read Distributed Traces When You Didn't Write the Code](/reading/2026-06/2026-06-10t223404-how-to-read-distributed-traces-when-you-didnt-write-the-code) treats observability as a reading skill, not just an instrumentation problem. Version control itself is in flux: [Reviewing Large Changes with Jujutsu](/reading/2026-05/2026-05-31t164252-reviewing-large-changes-with-jujutsu) and the [jj-vcs/jj](/reading/2026-05/2026-05-31t164554-jj-vcsjj) repository describe a Git-compatible VCS that auto-commits the working copy and treats conflicts as first-class objects, which changes how large PRs are reviewed.

The AI dimension cuts across everything. Several sources agree that speed gains are real and the quality risks are also real. [Agentic Coding is a Trap](/reading/2026-04/2026-04-27t145041-agentic-coding-is-a-trap) and [The Perils of AI to the Software Engineering Profession](/reading/2026-05/2026-05-14t223612-the-perils-of-ai-to-the-software-engineering-profession) both argue that full delegation to coding agents produces skill atrophy and ships code no one understands. [When Code Is Cheap, Does Quality Still Matter?](/reading/2026-05/2026-05-22t091746-when-code-is-cheap-does-quality-still-matter) answers yes: LLMs can generate polished technical debt faster than any individual engineer. [The Bottleneck Was Never the Code](/reading/2026-05/2026-05-06t110728-the-bottleneck-was-never-the-code) locates the real constraint in organizational alignment and specification clarity. [Code Smells When You Get AI to Write Your Frontend Tests](/reading/2026-06/2026-06-22t185420-code-smells-when-you-get-ai-to-write-your-frontend-tests) documents recurring AI-generated test failures including over-mocking and testing against buggy implementations rather than intended behavior. [How AI Code Review Can Make Correct Code Worse](/reading/2026-06/2026-06-23t212958-how-ai-code-review-can-make-correct-code-worse) finds that weaker fixer agents in review pipelines break correct code by overreaching beyond review scope. Against that, [Slow Mode](/reading/2026-05/2026-05-19t193626-slow-mode) proposes keeping the human involved at every agent step to preserve learning, and [Formal Methods and the Future of Programming](/reading/2026-06/2026-06-15t021106-formal-methods-and-the-future-of-programming) argues that agentic coding has made formal verification newly cost-effective.

Professional judgment receives the sharpest treatment in the knowledge-transfer sources. [The Tacit Dimension](/reading/2026-05/2026-05-19t110710-the-tacit-dimension-why-your-best-engineers-cant-tell-you) argues that the most valuable engineering expertise is structurally inaccessible to AI, transmissible only through apprenticeship. [Why Senior Developers Fail to Communicate Their Expertise](/reading/2026-05/2026-05-13t060018-why-senior-developers-fail-to-communicate-their-expertise) diagnoses a translation problem between engineering vocabulary (complexity management) and business vocabulary (uncertainty reduction). [Learn Algorithms for Interviews, Forget Them for Work](/reading/2026-04/2026-04-30t155134-learn-algorithms-for-interviews-forget-them-for-work) and [The Systemic Decay of Tech Hiring](/reading/2026-06/2026-06-22t182141-the-systemic-decay-of-tech-hiring) both show how hiring processes have drifted far from the skills that production engineering actually requires. [Your Onboarding Is a Hazing Ritual](/reading/2026-05/2026-05-08t112608-your-onboarding-is-a-hazing-ritual-and-you-call-it-agile) adds that once hired, new engineers face onboarding designed to surface failure rather than transmit knowledge.

Security sits at the intersection of craft and process. [Supply-Chain Attack Using Invisible Unicode Code](/reading/2026-04/2026-04-30t231634-supply-chain-attack-using-invisible-code-hits-github-and) describes 151 malicious npm packages that encoded payloads in invisible Unicode variation-selector characters, bypassing both code review and static analysis. [Approaching Zero Bugs](/reading/2026-05/2026-05-02t094735-approaching-zero-bugs) finds no measurable sign from curl's defect data that AI-assisted tooling is reducing latent bug counts in open-source projects.

The field is genuinely contested on what to optimize for. [The Software Engineering War](/reading/2026-07/2026-07-07t170607-the-software-engineering-war) names the split directly: builders prioritize shipping speed with AI, keepers insist on quality, and organizational context shapes which position any given engineer occupies. That tension does not resolve neatly, and the sources here do not pretend it does.
