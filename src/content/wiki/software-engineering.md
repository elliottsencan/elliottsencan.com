The practice of software engineering is less a single discipline than a collection of overlapping concerns: how code is structured, how systems are tested and deployed, how knowledge moves between people, and how new tools reshape the cost curve of producing working software.

On structure, a consistent theme across sources is that good design hides complexity behind small surfaces. The case for deep modules — small interfaces concealing large implementations — is made explicitly in [AI Likes Deep Modules](/reading/2026-05/2026-05-04t231343-ai-likes-deep-modules), which argues this property benefits human readers and LLM-assisted workflows equally. The Single Responsibility Principle points in the same direction but is routinely misread: [Single Responsibility, the Distorted Principle](/reading/2026-06/2026-06-04t073318-single-responsibility-the-distorted-principle) argues SRP is about cohesive grouping under one accountable concern, not atomizing code into the smallest conceivable unit. Angular component design runs into the same trap — [A Better Way to Build Angular Components](/reading/2026-04/2026-04-30t232001-a-better-way-to-build-angular-components-from-inputs-to) argues that components bloated with dozens of inputs should be decomposed into directives and sub-components so each responsibility is genuinely encapsulated.

Testing discipline shows up across several sources and the picture is mixed. [Designing Playwright Tests That Survive UI Refactors](/reading/2026-05/2026-05-05t135218-designing-playwright-tests-that-survive-ui-refactors) argues that brittle test suites fail because they couple to implementation details — CSS classes, DOM structure, position — rather than semantic roles. [Code Smells when you get AI to write your Frontend Tests](/reading/2026-06/2026-06-22t185420-code-smells-when-you-get-ai-to-write-your-frontend-tests) extends this, documenting over twenty AI-generated patterns including over-mocking and writing tests to match a buggy implementation rather than intended behavior. Meanwhile [What CI Actually Looks Like at a 100-Person Team](/reading/2026-04/2026-04-30t195531-what-ci-actually-looks-like-at-a-100-person-team) shows CI at scale — 575K weekly jobs and 33M test executions at PostHog — with an AI agent triaging flaky tests and opening fix PRs automatically. Despite tooling advances, [Approaching Zero Bugs](/reading/2026-05/2026-05-02t094735-approaching-zero-bugs) finds no measurable sign yet that projects are actually converging on zero latent defects.

The AI-assisted coding debate runs through many of these sources without resolution. [Agentic Coding is a Trap](/reading/2026-04/2026-04-27t145041-agentic-coding-is-a-trap) argues full agentic workflows accelerate skill atrophy and create vendor dependency. [Slow Mode](/reading/2026-05/2026-05-19t193626-slow-mode) proposes keeping the human in the loop at every step to preserve genuine learning. [The Perils of AI to the Software Engineering Profession](/reading/2026-05/2026-05-14t223612-the-perils-of-ai-to-the-software-engineering-profession) calls unreviewed AI-generated code categorically incompatible with safety-critical systems. On the other side, [Formal Methods and the Future of Programming](/reading/2026-06/2026-06-15t021106-formal-methods-and-the-future-of-programming) argues agentic coding makes formal verification newly cost-effective by lowering the cost of writing proofs. [The Coming Loop](/reading/2026-06/2026-06-23t161552-the-coming-loop) acknowledges harness loops around coding agents are becoming unavoidable but warns they amplify LLMs' worst tendencies — defensive, opaque code — and risk creating codebases that require machine participation to maintain. [When Code Is Cheap, Does Quality Still Matter](/reading/2026-05/2026-05-22t091746-when-code-is-cheap-does-quality-still-matter) answers yes: AI lowers production cost but not ownership cost, and LLMs can generate polished technical debt faster than any individual engineer.

The organizational layer matters at least as much as the technical one. [The Bottleneck Was Never the Code](/reading/2026-05/2026-05-06t110728-the-bottleneck-was-never-the-code) argues coding agents make individual code-writing cheap but the real bottleneck remains shared context, specification clarity, and management coherence. [The Tacit Dimension](/reading/2026-05/2026-05-19t110710-the-tacit-dimension-why-your-best-engineers-cant-tell-you) draws on Polanyi to argue that the most valuable engineering expertise — pattern recognition, design intuition, unwritten conventions — is structurally inaccessible to AI and can only be transmitted through apprenticeship. [Why Senior Developers Fail to Communicate Their Expertise](/reading/2026-05/2026-05-13t060018-why-senior-developers-fail-to-communicate-their-expertise) locates a different gap: engineers speak in terms of complexity management while the business thinks in uncertainty reduction. [Your Onboarding Is a Hazing Ritual](/reading/2026-05/2026-05-08t112608-your-onboarding-is-a-hazing-ritual-and-you-call-it-agile) shows how dysfunctional processes set new hires up to fail while remaining invisible to management.

Hiring itself is under strain. [Learn Algorithms for Interviews, Forget Them for Work](/reading/2026-04/2026-04-30t155134-learn-algorithms-for-interviews-forget-them-for-work) argues algorithm interviews test a narrow trainable skill that weakly correlates with production performance. [The Systemic Decay of Tech Hiring](/reading/2026-06/2026-06-22t182141-the-systemic-decay-of-tech-hiring) traces how error asymmetry drives ever-harder tests and candidates overfit to the process via Goodhart's Law.

Tooling and platform reliability round out the picture. [The Unwritten Laws of Software Engineering](/reading/2026-06/2026-06-10t073045-the-unwritten-laws-of-software-engineering) distills rules from real production incidents, including rolling back before debugging and treating every external dependency as a future outage. [GitHub is Sinking](/reading/2026-05/2026-05-10t205349-github-is-sinking) argues GitHub's reliability has declined enough that engineers should evaluate alternatives. [Supply-chain attack using invisible Unicode](/reading/2026-04/2026-04-30t231634-supply-chain-attack-using-invisible-code-hits-github-and) shows that even static analysis can miss malicious payloads encoded in invisible characters. Performance optimization remains context-dependent: [5x Faster fast_blur in image-rs](/reading/2026-05/2026-05-14t151252-5-faster-fastblur-in-image-rs) demonstrates algorithmic gains through integer accumulators and reciprocal multiplication, while [When Impressive Performance Gains Do Not Matter](/reading/2026-06/2026-06-30t185207-when-impressive-performance-gains-do-not-matter) explains why even order-of-magnitude improvements can fail to change outcomes under attention thresholds, discrete capacity increments, or pipeline backpressure.
