Software engineering resists a single clean definition because it spans so many layers: the design decisions inside a module, the tooling that supports a team, the processes that get code from editor to production, and the organizational knowledge that makes any of it transferable. The sources here collectively push against reductive framings of the discipline — algorithm fluency as a hiring proxy, agentic AI as a productivity shortcut, or any single principle applied without judgment.

At the code level, good design means managing complexity, not minimizing line count. [AI Likes Deep Modules](/reading/2026-05/2026-05-04t231343-ai-likes-deep-modules) argues that small interfaces hiding large implementations reduce cognitive load for both humans and LLMs. [Single Responsibility, the Distorted Principle](/reading/2026-06/2026-06-04t073318-single-responsibility-the-distorted-principle) makes a related point: SRP is about cohesive accountability, not atomization. Over-granularizing classes produces the very fragmentation the principle is supposed to prevent. The Idiot Index for Code](/reading/2026-06/2026-06-22t000701-the-idiot-index-for-code) applies a cost-of-complexity lens to bloated engineering, treating over-engineered code as a signal of low-value work.

Practical craft extends well beyond architecture patterns. [Shell Tricks That Actually Make Life Easier](/reading/2026-04/2026-04-30t231815-shell-tricks-that-actually-make-life-easier-and-save-your) catalogs underused shell shortcuts that reduce friction at the command line. [Using SSH Keys](/reading/2026-05/2026-05-04t231548-using-ssh-keys-to-make-connectivity-simpler-and-secure) covers key generation and agent forwarding as baseline security hygiene for working across remote machines. [The Git Commands I Run Before Reading Any Code](/reading/2026-06/2026-06-18t024208-the-git-commands-i-run-before-reading-any-code) treats git log history as diagnostic data — churn hotspots, bus factor, and firefighting frequency — before a single file is opened. [Reviewing Large Changes with Jujutsu](/reading/2026-05/2026-05-31t164252-reviewing-large-changes-with-jujutsu) demonstrates a concrete workflow for large pull requests using Jujutsu's first-class conflict model to persist review progress in version control itself.

Testing and reliability form another persistent theme. [Designing Playwright Tests That Survive UI Refactors](/reading/2026-05/2026-05-05t135218-designing-playwright-tests-that-survive-ui-refactors) argues that test fragility comes from coupling to implementation details rather than semantic roles. [What CI Actually Looks Like at a 100-Person Team](/reading/2026-04/2026-04-30t195531-what-ci-actually-looks-like-at-a-100-person-team) describes AI-assisted triage at PostHog's scale — 575K weekly jobs — as an operational necessity, not a luxury. [Approaching Zero Bugs](/reading/2026-05/2026-05-02t094735-approaching-zero-bugs) uses curl's vulnerability history to argue there is no measurable evidence yet that AI-assisted static analysis is closing the latent bug gap. [Testing a Security Tool Like It Can Hurt People](/reading/2026-06/2026-06-11t024225-testing-a-security-tool-like-it-can-hurt-people) goes further, building a deterministic assurance platform with fixture invariants and red runs that prove the system fails loudly rather than over-claiming certainty.

Security is not only about code correctness. [Supply-chain attack using invisible Unicode code](/reading/2026-04/2026-04-30t231634-supply-chain-attack-using-invisible-code-hits-github-and) documents 151 malicious npm packages that encoded payloads in invisible variation-selector characters, bypassing both human reviewers and static analysis. [YAML? That's Norway Problem](/reading/2026-05/2026-05-18t113714-yaml-thats-norway-problem) traces a decade-old spec bug still present in widely used libraries, illustrating how configuration formats carry silent risks.

On hiring and expertise, [Learn Algorithms for Interviews, Forget Them for Work](/reading/2026-04/2026-04-30t155134-learn-algorithms-for-interviews-forget-them-for-work) makes the case that algorithm interviews test a narrow trainable skill weakly correlated with production performance. [Why Senior Developers Fail to Communicate Their Expertise](/reading/2026-05/2026-05-13t060018-why-senior-developers-fail-to-communicate-their-expertise) identifies a translation problem: senior engineers frame value in terms of complexity management while the business thinks in terms of uncertainty reduction. [The Tacit Dimension](/reading/2026-05/2026-05-19t110710-the-tacit-dimension-why-your-best-engineers-cant-tell-you) draws on Polanyi to argue that the most valuable engineering knowledge — pattern recognition, design intuition, unwritten conventions — is structurally inaccessible to documentation or AI tools, and survives only through apprenticeship.

Team dynamics shape outcomes as much as individual skill. [Your Onboarding Is a Hazing Ritual](/reading/2026-05/2026-05-08t112608-your-onboarding-is-a-hazing-ritual-and-you-call-it-agile) describes how dysfunction masquerading as agile process systematically sets new hires up to fail. [The Bottleneck Was Never the Code](/reading/2026-05/2026-05-06t110728-the-bottleneck-was-never-the-code) argues that coding agents amplify whatever organizational alignment or misalignment already exists; shared context and specification clarity remain the real constraints. [The Unwritten Laws of Software Engineering](/reading/2026-06/2026-06-10t073045-the-unwritten-laws-of-software-engineering) distills production-incident lessons — roll back before debugging, treat every external dependency as a future outage — that no amount of tooling replaces.

The AI dimension runs through nearly everything. [Agentic Coding is a Trap](/reading/2026-04/2026-04-27t145041-agentic-coding-is-a-trap) warns that full agentic workflows accelerate skill atrophy and create vendor dependency. [The Perils of AI to the Software Engineering Profession](/reading/2026-05/2026-05-14t223612-the-perils-of-ai-to-the-software-engineering-profession) argues vibe coding is categorically incompatible with safety-critical systems. [When Code Is Cheap, Does Quality Still Matter](/reading/2026-05/2026-05-22t091746-when-code-is-cheap-does-quality-still-matter) answers yes: AI lowers production cost but not ownership cost, and taste and judgment remain necessary to avoid generating polished technical debt at scale. Against that, [Slow Mode](/reading/2026-05/2026-05-19t193626-slow-mode) proposes a human-in-the-loop agent that trades short-term speed for genuine learning, and [Formal Methods and the Future of Programming](/reading/2026-06/2026-06-15t021106-formal-methods-and-the-future-of-programming) argues agentic coding has made formal verification newly cost-effective by lowering the cost of writing proofs while creating urgent demand for verification beyond what tests provide.
