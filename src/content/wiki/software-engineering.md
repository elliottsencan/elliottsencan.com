Software engineering is not a single discipline but a cluster of overlapping concerns: writing correct code, designing systems that survive contact with reality, hiring and onboarding people who can maintain those systems, and communicating technical decisions to organizations that think in different terms. The sources here span all of those layers.

At the implementation level, several pieces address what good code actually looks like. The deep-module principle — small interfaces hiding large implementations — reduces cognitive load for both human readers and LLMs attempting to extend the code [AI Likes Deep Modules](/reading/2026-05/2026-05-04t231343-ai-likes-deep-modules). The Single Responsibility Principle is commonly misread as "do one thing"; the more precise formulation is cohesive grouping under a single accountable concern, and over-granularizing classes violates the cognitive simplicity SRP is supposed to deliver [Single Responsibility, the Distorted Principle](/reading/2026-06/2026-06-04t073318-single-responsibility-the-distorted-principle). Angular component design exhibits the same failure mode when dozens of inputs accumulate on a single component; the Composite Components pattern distributes that surface area across directives and sub-components [A Better way to build Angular Components](/reading/2026-04/2026-04-30t232001-a-better-way-to-build-angular-components-from-inputs-to).

At the system and infrastructure level, the picture is similarly practical. Continuous integration at scale requires automation that goes beyond running tests: at PostHog's 575K weekly jobs, an AI triage agent ingests billions of log lines and opens fix PRs for flaky tests automatically [What CI Actually Looks Like at a 100-Person Team](/reading/2026-04/2026-04-30t195531-what-ci-actually-looks-like-at-a-100-person-team). Container filesystem isolation — how mount namespaces, mount propagation, and pivot_root compose — is worth understanding from first principles before relying on Docker abstractions [How Container Filesystem Works](/reading/2026-05/2026-05-04t231858-how-container-filesystem-works-building-a-docker-like). Distributed traces let engineers read systems they did not write: span anatomy, critical-path analysis, and N+1 staircase patterns provide a vocabulary for diagnosing unfamiliar codebases [How to read distributed traces when you didn't write the code](/reading/2026-06/2026-06-10t223404-how-to-read-distributed-traces-when-you-didnt-write-the-code). Version control tooling continues to evolve; Jujutsu's auto-committed working copy and first-class conflict objects change how large reviews can be structured without losing progress [Reviewing Large Changes with Jujutsu](/reading/2026-05/2026-05-31t164252-reviewing-large-changes-with-jujutsu).

Security is not separate from engineering practice. A supply-chain attack embedding payloads in invisible Unicode variation-selector characters passed undetected through code review and static analysis [Supply-chain attack using invisible Unicode code](/reading/2026-04/2026-04-30t231634-supply-chain-attack-using-invisible-code-hits-github-and). Separately, YAML's Norway problem — where `NO` parses as false — persists in production libraries a decade after the spec was fixed, a reminder that parser behavior and spec correctness diverge in practice [YAML? That's Norway Problem](/reading/2026-05/2026-05-18t113714-yaml-thats-norway-problem). Seven production-hardened rules — roll back before debugging, treat every dependency as a future outage — sit in the same category: operational wisdom that static analysis cannot encode [The Unwritten Laws of Software Engineering](/reading/2026-06/2026-06-10t073045-the-unwritten-laws-of-software-engineering).

The profession's relationship with AI is the most contested layer. Multiple sources converge on a core tension: AI lowers the cost of producing code but not the cost of owning it. Technical debt generated at LLM speed accumulates faster than any individual engineer ever could [When Code Is Cheap, Does Quality Still Matter?](/reading/2026-05/2026-05-22t091746-when-code-is-cheap-does-quality-still-matter). Full agentic workflows accelerate skill atrophy and invert developer priorities toward speed over understanding [Agentic Coding is a Trap](/reading/2026-04/2026-04-27t145041-agentic-coding-is-a-trap). The argument for a "Slow Mode" agent — one that plans alongside the programmer rather than looping autonomously — is that it trades short-term throughput for genuine comprehension and long-term ownership [Slow Mode](/reading/2026-05/2026-05-19t193626-slow-mode). Vibe coding without review or testing is categorically incompatible with safety-critical systems [The Perils of AI to the Software Engineering Profession](/reading/2026-05/2026-05-14t223612-the-perils-of-ai-to-the-software-engineering-profession). Formal methods become more cost-effective in this environment, both because agentic tools lower proof-writing costs and because the verification gap that tests alone cannot close grows wider [Formal Methods and the Future of Programming](/reading/2026-06/2026-06-15t021106-formal-methods-and-the-future-of-programming).

Organizational concerns run through multiple sources. Coding agents amplify whatever alignment or misalignment already exists in a team; the real bottleneck was always shared context and specification clarity [The bottleneck was never the code](/reading/2026-05/2026-05-06t110728-the-bottleneck-was-never-the-code). Senior engineers communicate in terms of complexity management while the rest of the business thinks in terms of uncertainty reduction, and bridging that gap is a distinct skill from writing good code [Why Senior Developers Fail to Communicate Their Expertise](/reading/2026-05/2026-05-13t060018-why-senior-developers-fail-to-communicate-their-expertise). The most valuable engineering expertise — pattern recognition, design intuition, unwritten conventions — is structurally inaccessible to AI tools and can only be transmitted through apprenticeship [The Tacit Dimension](/reading/2026-05/2026-05-19t110710-the-tacit-dimension-why-your-best-engineers-cant-tell-you). Onboarding practices that disguise hazing as agile process compound these knowledge-transfer failures [Your Onboarding Is a Hazing Ritual and You Call It Agile](/reading/2026-05/2026-05-08t112608-your-onboarding-is-a-hazing-ritual-and-you-call-it-agile).

Algorithm interview performance weakly correlates with production engineering quality; real engineering requires reading tradeoffs, shipping incrementally, and handling messy unbounded inputs [Learn Algorithms for Interviews, Forget Them for Work](/reading/2026-04/2026-04-30t155134-learn-algorithms-for-interviews-forget-them-for-work). Hiring processes themselves have decayed through well-intentioned error-asymmetry: interviewers made tests progressively harder to avoid false positives, candidates overfit via Goodhart's Law, and accountability dissolved into shared interviewer pools [The systemic decay of tech hiring](/reading/2026-06/2026-06-22t182141-the-systemic-decay-of-tech-hiring).

Across all these layers, a consistent thread runs: tools change faster than judgment, and judgment is what actually determines whether a system survives in production.
