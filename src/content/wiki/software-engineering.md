Software engineering is not a single discipline so much as a cluster of overlapping concerns: how code is designed, how it is verified, how it is maintained over time, and how the humans writing it learn and coordinate. The sources collected here illuminate each of these in ways that push against each other productively.

On design, the principle most frequently distorted is the Single Responsibility Principle. [Henrique Teixeira argues](/reading/2026-06/2026-06-04t073318-single-responsibility-the-distorted-principle) that SRP is misread as "do only one thing" when Robert Martin's original formulation concerns cohesive grouping under a single accountable responsibility — over-granularizing classes violates the very cognitive simplicity SRP is meant to provide. A complementary argument comes from module depth: [Go Monk contends](/reading/2026-05/2026-05-04t231343-ai-likes-deep-modules) that small interfaces hiding large implementations reduce complexity for both human readers and LLMs, contrasting shallow modules that leak implementation detail with deep ones that absorb it.

Testing is where the gap between theory and production practice shows most sharply. [Fagner Brack argues](/reading/2026-04/2026-04-30t155134-learn-algorithms-for-interviews-forget-them-for-work) that algorithm interviews test a narrow, trainable skill weakly correlated with real engineering ability; production work demands reading tradeoffs and shipping incrementally against messy, unbounded inputs. On the testing infrastructure side, [Sam Alba describes](/reading/2026-04/2026-04-30t195531-what-ci-actually-looks-like-at-a-100-person-team) Mendral's AI-driven CI triage at PostHog handling 575K weekly jobs and 33M test executions, automatically tracing flaky tests to root causes and opening fix PRs — a scale at which manual triage becomes impractical. Meanwhile, [the Currents team shows](/reading/2026-05/2026-05-05t135218-designing-playwright-tests-that-survive-ui-refactors) that Playwright suites break during refactors because they couple to implementation details rather than semantic roles and accessible names, and [extends this](/reading/2026-05/2026-05-15t120337-playwright-testing-in-staging-vs-production) with a framework for splitting tests between staging and production environments.

Security has grown more treacherous. [Dan Goodin reports](/reading/2026-04/2026-04-30t231634-supply-chain-attack-using-invisible-code-hits-github-and) that attackers encoded payloads in invisible Unicode variation-selector characters across 151 malicious npm and GitHub packages — undetectable by code review and static analysis, yet executable at runtime. [Daniel Stenberg's analysis of curl's bug history](/reading/2026-05/2026-05-02t094735-approaching-zero-bugs) finds no measurable sign that AI-assisted static analysis is moving open-source projects toward zero latent defects, despite the power of new tooling. [Yaron Minsky at Jane Street](/reading/2026-06/2026-06-15t021106-formal-methods-and-the-future-of-programming) takes the opposite view on the longer arc: agentic coding has made formal methods newly cost-effective by lowering proof-writing costs while simultaneously creating urgent demand for verification that goes beyond tests.

The AI-assisted coding debate is where the most contested ground lies. [Lars Faye argues](/reading/2026-04/2026-04-27t145041-agentic-coding-is-a-trap) that full agentic workflows accelerate skill atrophy, invert developer priorities toward speed over understanding, and create vendor dependency — LLMs should remain secondary delegation tools while humans stay hands-on. [Abednego Gomes extends this](/reading/2026-05/2026-05-14t223612-the-perils-of-ai-to-the-software-engineering-profession) to a categorical claim: shipping unreviewed AI-generated code is reckless and incompatible with safety-critical systems. [Pete Millspaugh's Slow Mode proposal](/reading/2026-05/2026-05-19t193626-slow-mode) sits in the same camp, arguing for an agent design that keeps humans involved at every planning step to trade short-term productivity for genuine learning. [Yusuf Aytas puts a cost-accounting frame on it](/reading/2026-05/2026-05-22t091746-when-code-is-cheap-does-quality-still-matter): AI lowers the cost of producing code but not of owning it — LLMs generate polished technical debt faster than any individual engineer ever could.

Organizational concerns cut across all of this. [The Typical Set observes](/reading/2026-05/2026-05-06t110728-the-bottleneck-was-never-the-code) that coding agents make individual code-writing cheap but the real bottleneck was always shared context, specification clarity, and management coherence — agents amplify existing organizational alignment or misalignment. [Tuhin Nair identifies](/reading/2026-05/2026-05-13t060018-why-senior-developers-fail-to-communicate-their-expertise) a language mismatch: senior engineers communicate in terms of complexity management while the business thinks in terms of uncertainty reduction. [cekrem draws on Polanyi](/reading/2026-05/2026-05-19t110710-the-tacit-dimension-why-your-best-engineers-cant-tell-you) to argue that the most valuable engineering expertise — pattern recognition, design intuition, unwritten conventions — is structurally inaccessible to AI tools and can only be transmitted through apprenticeship. Poor onboarding compounds this: [DHg documents](/reading/2026-05/2026-05-08t112608-your-onboarding-is-a-hazing-ritual-and-you-call-it-agile) how packed calendars, same-sprint workloads, and probation-enforced silence disguise dysfunction as agile process.

At the tooling and infrastructure layer, the collection covers version control ([Jujutsu's first-class conflict model](/reading/2026-05/2026-05-31t164554-jj-vcsjj), [review workflows for large changes](/reading/2026-05/2026-05-31t164252-reviewing-large-changes-with-jujutsu)), [shell scripting safeguards and Readline bindings](/reading/2026-04/2026-04-30t231815-shell-tricks-that-actually-make-life-easier-and-save-your), [SSH key authentication across remote machines](/reading/2026-05/2026-05-04t231548-using-ssh-keys-to-make-connectivity-simpler-and-secure), and [container filesystem isolation built from Linux primitives](/reading/2026-05/2026-05-04t231858-how-container-filesystem-works-building-a-docker-like). [Anton Zaides distills production lessons](/reading/2026-06/2026-06-10t073045-the-unwritten-laws-of-software-engineering) into rules like rolling back before debugging and treating every external dependency as a future outage.

The through-line across all of it is that engineering quality is not reducible to code volume or tool capability. Whether the concern is module depth, test stability, supply-chain integrity, or skill transmission, the durable challenges are ones of judgment, communication, and organizational structure — properties that compound or erode independently of how fast the code gets written.
