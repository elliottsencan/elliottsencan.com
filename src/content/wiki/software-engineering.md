Software engineering resists a clean definition because it spans at least three distinct concerns: the technical craft of writing and structuring code, the organizational and process work of shipping systems reliably, and the professional judgment that distinguishes engineers who understand what they build from those who merely produce it.

On the craft side, sources here cover a wide range of implementation-level topics. [Crafting Interpreters](/reading/2026-04/2026-04-30t231027-munificentcraftinginterpreters) demonstrates that building complete, working systems from scratch remains one of the most durable ways to develop genuine understanding. The [5x fast_blur optimization](/reading/2026-05/2026-05-14t151252-5-faster-fastblur-in-image-rs) shows the same principle at the algorithmic level: replacing float arithmetic with integer accumulators and reciprocal multiplication to achieve a 5.9x speedup is the kind of reasoning no linter or AI can substitute for. [Shell tricks](/reading/2026-04/2026-04-30t231815-shell-tricks-that-actually-make-life-easier-and-save-your) and [SSH key management](/reading/2026-05/2026-05-04t231548-using-ssh-keys-to-make-connectivity-simpler-and-secure) occupy the same space: fluency with the environment an engineer works in every day.

Design principles run through multiple sources with more friction than consensus. The [Single Responsibility Principle](/reading/2026-06/2026-06-04t073318-single-responsibility-the-distorted-principle) is widely misread as "do only one thing" when it actually means grouping behaviors under a single accountable responsibility; over-granularizing classes violates the cognitive simplicity SRP is meant to provide. [Deep modules](/reading/2026-05/2026-05-04t231343-ai-likes-deep-modules) offer a complementary view: small interfaces hiding large implementations reduce complexity for both humans and LLMs. [Angular component composition](/reading/2026-04/2026-04-30t232001-a-better-way-to-build-angular-components-from-inputs-to) applies related thinking to UI architecture, arguing that components bloated with dozens of inputs should be refactored into directives and sub-components.

Testing and reliability surface repeatedly. [Playwright test design](/reading/2026-05/2026-05-05t135218-designing-playwright-tests-that-survive-ui-refactors) argues that suites break during refactors not because of bad selector choices but because tests couple to implementation details rather than semantic roles and accessible names. [CI at scale](/reading/2026-04/2026-04-30t195531-what-ci-actually-looks-like-at-a-100-person-team) shows what automation looks like when it actually works: an AI agent triaging 575K weekly jobs and opening fix PRs for flaky tests. Daniel Stenberg's analysis of curl [finds no measurable evidence](/reading/2026-05/2026-05-02t094735-approaching-zero-bugs) that even powerful AI-assisted static analysis is pushing open-source projects toward zero latent bugs. The [unwritten laws of software engineering](/reading/2026-06/2026-06-10t073045-the-unwritten-laws-of-software-engineering) distill the same intuition from production incidents: roll back before debugging, treat every external dependency as a future outage.

Observability and code archaeology get practical treatment too. [Distributed traces](/reading/2026-06/2026-06-10t223404-how-to-read-distributed-traces-when-you-didnt-write-the-code) provide a method for reading unfamiliar systems through span anatomy and critical-path analysis. [Git log commands](/reading/2026-06/2026-06-18t024208-the-git-commands-i-run-before-reading-any-code) reveal a codebase's risks before opening a single file, and a [Rails audit process](/reading/2026-06/2026-06-18t090801-how-i-audit-a-legacy-rails-codebase-in-the-first-week) starts with stakeholder interviews rather than tools.

The AI-and-craft tension is where the sources generate the most disagreement. [Lars Faye](/reading/2026-04/2026-04-27t145041-agentic-coding-is-a-trap) argues that full agentic workflows accelerate skill atrophy and invert priorities toward speed over understanding. [Abednego Gomes](/reading/2026-05/2026-05-14t223612-the-perils-of-ai-to-the-software-engineering-profession) puts this more sharply: shipping AI-generated code without review is incompatible with safety-critical systems. [Yusuf Aytas](/reading/2026-05/2026-05-22t091746-when-code-is-cheap-does-quality-still-matter) adds that AI lowers the cost of producing code but not the cost of owning it, making taste and judgment more important, not less. Against this, [Val Town's Slow Mode proposal](/reading/2026-05/2026-05-19t193626-slow-mode) suggests keeping humans in every agent planning loop, trading short-term speed for genuine learning. [The Typical Set](/reading/2026-05/2026-05-06t110728-the-bottleneck-was-never-the-code) frames the same tension organizationally: coding agents amplify existing alignment or misalignment because the bottleneck was never code production.

Professional and organizational dimensions close the picture. [Algorithm interviews](/reading/2026-04/2026-04-30t155134-learn-algorithms-for-interviews-forget-them-for-work) test a narrow, trainable skill that weakly correlates with production performance; real engineering requires reading tradeoffs and shipping incrementally. [Senior developers](/reading/2026-05/2026-05-13t060018-why-senior-developers-fail-to-communicate-their-expertise) communicate in terms of complexity management while the rest of the business thinks in uncertainty reduction. [Tacit knowledge](/reading/2026-05/2026-05-19t110710-the-tacit-dimension-why-your-best-engineers-cant-tell-you) is the sharpest version of this claim: the most valuable engineering expertise is structurally inaccessible to AI tools and can only be transmitted through apprenticeship. [Poor onboarding](/reading/2026-05/2026-05-08t112608-your-onboarding-is-a-hazing-ritual-and-you-call-it-agile) systematically destroys the conditions under which that transmission can happen.

Security belongs in the same picture. A [supply-chain attack](/reading/2026-04/2026-04-30t231634-supply-chain-attack-using-invisible-code-hits-github-and) encoding payloads in invisible Unicode characters made malicious packages undetectable by code reviewers and static analysis alike, which is a reminder that engineering judgment must include threat modeling at every layer of the stack. [Formal methods](/reading/2026-06/2026-06-15t021106-formal-methods-and-the-future-of-programming) are becoming newly cost-effective as agentic coding creates demand for verification that goes beyond what tests alone can provide.
