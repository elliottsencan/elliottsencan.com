## Reading / Listening

A few things from the recent read pile worth naming. Marc Brooker's piece on Nagle's algorithm is a good reminder that latency bugs hide in defaults nobody questions anymore. The frontend history walkthrough traces the 44-layer stack back to the specific pain each tool was built to solve, which is a more honest accounting than most retrospectives manage.

On the AI engineering side: Armin Ronacher's "The Coming Loop" is still the piece I keep returning to. His argument that outer harness loops amplify LLMs' worst tendencies and risk producing codebases that require machine participation to maintain feels more relevant every week. The Imbue work on AI code review regressions is a useful companion read: weaker fixer agents overreach beyond review scope and quietly break correct code.

The builders-vs-keepers framing in "The Software Engineering War" is a decent lens on a real split, even if the resolution is a bit soft.
